# Codex config layers

Codex runtime configuration is generated by `configctl` from this layered source tree.

```text
~/.config/codex/
├── adopted.d/   # dotfiles-managed fragments adopted from previous runtime config
├── custom.d/    # promoted custom fragments
└── local.conf   # machine-local overrides, not managed here
```

The generated runtime output is:

```text
~/.codex/config.toml
```

Do not manage `~/.codex/config.toml` directly with Home Manager. It must remain writable so Codex and `configctl` can reconcile mutable user state such as project trust.
