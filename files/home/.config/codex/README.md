# Codex config layers

Codex runtime config is generated by `configctl` from these source layers.

```text
~/.config/codex/
├── local.conf
├── custom.d/
└── adopted.d/
```

`~/.codex/config.toml` is runtime output and should not be managed directly by Home Manager.

- `adopted.d/` contains repo-managed fragments.
- `custom.d/` is for local fragments that may be promoted later.
- `local.conf` is machine-local and should not be committed as live state.
